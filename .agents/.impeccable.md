## Project Context

### What imgstry-ui is
A Svelte 5 desktop-class photo-editor UI that drives the `imgstry` engine.
Sliders, curve editor, color picker, histogram, tone-curve — wired into a
single live preview viewport. Runs entirely in the browser; no server.

### Design Goals
- **Live, responsive editing.** Every adjustment renders through the imgstry
  pipeline with a small debounce; the viewport stays smooth.
- **Engine as the single source of truth.** UI state describes adjustments;
  the engine applies them. The UI never re-implements pixel math.
- **Pro-tool feel.** Clear hierarchy: rails frame the workspace, the viewport
  is the centerpiece, controls feel precise rather than playful.
- **Keyboard-friendly.** Power-user shortcuts (preview-original, reset) are
  first-class.

### Non-goals
- No backend. No accounts. No cloud sync.
- No mobile-first layout (a small-screen pass can come later).
- No custom pixel ops in the UI — that belongs in `imgstry`.

### Architecture in one breath
`src/App.svelte` stitches `TopBar` / `LeftRail` / `RightRail` / `Viewport`
around a single editor state object (`src/lib/editor/editor.svelte.ts`).
The editor holds the source image, the adjustment record, history, and
delegates rendering to the imgstry pipeline.
